# rtiprep

`rtiprep` is a fast, command-line utility written in Go for processing and preparing Reflectance Transformation Imaging (RTI) files, such as Polynomial Texture Maps (PTM) and Hemispherical Harmonics (HSH), as well as standard images for web-based multi-resolution visualization.

It supports slicing multi-layer images into hierarchical image pyramid tiles (JPEG or PNG) or packing them into a single, tiled pyramidal TIFF (Cloud Optimized GeoTIFF-like format) with embedded spatial/luminance coefficient metadata.

## Web Viewer Compatibility

The outputs generated by `rtiprep` are designed to be served statically and rendered in the browser. They are fully compatible with:
- [modernRtiViewer](https://github.com/mfindeisen/modernRtiViewer) - A modern WebGL-based viewer for Reflectance Transformation Imaging (RTI) data, supporting both image-pyramid folders and tiled pyramidal TIFFs directly in the browser.


## Features

- **Format Support:** Input formats include `.ptm`, `.rti`, `.jpg`, `.jpeg`, `.png`, `.tif`, and `.tiff`.
- **Pyramid Tiling:** Recursively divides layers into standard tiles (default 256x256 px) with 1-pixel neighbor borders (seam border padding) to ensure smooth blending at tile boundaries in WebGL.
- **Pyramidal TIFF Export:** Combines all layers as chunky interleaved channels in a single pyramidal TIFF file.
- **Neural RTI Integration:** Embed decoder neural network weights directly into the pyramidal TIFF metadata (`ImageDescription` tag) for GeoTIFF-based web renderers.
- **Parallel Processing:** Highly concurrent execution leveraging Go's goroutines.

## Installation

Ensure you have [Go](https://go.dev/) (version 1.18 or higher recommended) installed.

Clone the repository and build the binary:

```bash
git clone <repository-url>
cd rtiprep
go build -o rtiprep
```

## Usage

```bash
./rtiprep [options] <input_file>
```

### Options

| Flag | Default | Description |
|---|---|---|
| `-t <size>` | `256` | Size of the output tiles in pixels. |
| `-q <quality>` | `90` | Quality of saved JPEG tiles (1-100). |
| `-p` | `false` | Save output tiles as PNG instead of JPEG. |
| `-o <path>` | *Auto* | Output destination directory (or file path if using `-tiff`). Defaults to `<input_name_without_ext>`. |
| `-tiff` | `false` | Save output as a single, tiled pyramidal TIFF file instead of a folder of images. |
| `-weights <path>` | `""` | *(Optional)* Path to a Neural RTI decoder weights JSON file to embed in the TIFF metadata. |

### Examples

**1. Slice a PTM file into a folder of JPEG tiles (default):**
```bash
./rtiprep object.ptm
```
This generates a folder named `object` containing all sliced tiles and an `info.json` descriptor file.

**2. Slice a file into PNG tiles with a custom tile size:**
```bash
./rtiprep -p -t 512 -o output_tiles object.rti
```

**3. Generate a single Pyramidal TIFF file:**
```bash
./rtiprep -tiff object.rti
```
This creates a single pyramidal file named `object.tif`.

**4. Generate a Pyramidal TIFF with embedded Neural RTI weights:**
```bash
./rtiprep -tiff -weights weights.json object.png
```
